# Sim-3v3-Simple-Framework 新手入门指南

> 3v3 足球仿真机器人策略框架 | 基于 Booster Agent Framework | Python

---

## 1. 项目简介

这是一个 **3v3 足球仿真机器人策略框架**，用于在 SoccerSim 仿真环境中控制 3 个机器人进行足球比赛。框架采用三层架构设计，运行频率 30Hz，基于 Python 编写。

**核心设计理念：**
- **低上手门槛**：策略逻辑集中在一个文件（`main.py`），参数集中在一个文件（`param.py`）
- **职责清晰**：策略层决定"做什么"，动作层定义"怎么做"，执行层负责"怎么执行"
- **灵活扩展**：可以轻松添加新动作、修改站位、调整策略

简单来说：**你写策略，框架帮你跑比赛。**

---

## 2. 项目是做什么的

在 SoccerSim 仿真环境中，每支队伍由 3 个机器人组成，进行 3v3 的足球比赛。本项目就是控制己方 3 个机器人的「大脑」—— 决定每个机器人在什么时候跑向哪里、什么时候踢球、什么时候防守。

**核心能力：**
- 根据比赛阶段（开球、正常比赛、定位球等）自动分派球员角色
- 进攻：追球、射门
- 防守：守门、卡位
- 支援：跑位接应
- 避障：自动绕开其他机器人和球门结构
- 路径规划：A* 全局规划 + VFH 局部规划

---

## 3. 三层架构详解

框架采用清晰的三层架构，每一层职责明确，向下调用：

```
┌──────────────────────────────────────────────────────┐
│  main.py          策略层 — 决定"做什么"                │
│  ┌──────────────────────────────────────────────────┐│
│  │ Phase 状态机 → 选择进攻者 → 分配角色              ││
│  │ NORMAL: attack / guard / support                 ││
│  │ KICKOFF: take_kickoff + walk_to_slots            ││
│  │ READY: walk_to_slots                             ││
│  └──────────────────────────────────────────────────┘│
│                        ↓ 调用                        │
│  player.py        动作层 — 定义"怎么做"                │
│  ┌──────────────────────────────────────────────────┐│
│  │ attack()       追球 + 射门                        ││
│  │ guard()        回到球门区守门                      ││
│  │ support()      跑位到球后方支援                    ││
│  │ take_kickoff() 开球动作（走到球后+踢球）           ││
│  │ walk_to_slots() 批量分配站位（最优匹配）           ││
│  └──────────────────────────────────────────────────┘│
│                        ↓ 调用                        │
│  motion.py        执行层 — 负责"怎么执行"              │
│  ┌──────────────────────────────────────────────────┐│
│  │ walk_to()      行走至目标（支持避障）              ││
│  │ face_to()      原地转向                           ││
│  │ kick()         执行踢球                           ││
│  │ ensure_ready() 确保机器人就绪                     ││
│  │ set_velocity() 底层速度指令                        ││
│  └──────────────────────────────────────────────────┘│
└──────────────────────────────────────────────────────┘
```

**数据流向：**
每帧（30Hz），框架构造一个 `Context` 快照（包含球的位置、所有机器人位置、比赛状态），传入 `play()` 函数。策略层根据 Context 决定每个球员的动作，逐层向下调用，最终通过 SDK 发送速度/踢球指令给机器人。

## 5. 核心概念

### 5.1 Context（上下文）

每帧由框架构造的只读快照，是策略决策的唯一数据来源。`context` 在 player.py 中是框架自动注入的，每帧更新。

```python
context.now          # 当前时间戳
context.dt           # 距上一帧的时间间隔
context.team_id      # 己方队伍编号（1 或 2）
context.field        # 场地尺寸（含 own_goal, opponent_goal, half_length 等属性）
context.game         # 比赛控制状态（阶段、得分、罚牌等）
context.ball         # 球的位置
context.teammates    # 已方队友位置 {player_id: RobotState}
context.opponents    # 对方机器人位置 {player_id: RobotState}
context.pre_context  # 上一帧的 Context（用于跨帧状态记忆）
```

**context.game 的子属性：**
```python
context.game.phase               # 当前比赛阶段（Phase 枚举）
context.game.strategy_state      # 策略跨帧状态容器（SimpleNamespace）
context.game.kicking_team        # 开球队伍 ID
context.game.state               # 比赛状态（PLAYING/STOPPED 等）
context.game.set_play            # 定位球类型（SetType 枚举）
```

**strategy_state 用法：** 存储需要跨帧保持的状态，如进攻者 ID、开球者 ID：
```python
# 设置
context.game.strategy_state.normal_attacker = player.id
context.game.strategy_state.kickoff_taker = player.id

# 读取
preferred_id = context.game.strategy_state.normal_attacker
```

### 5.2 Phase（比赛阶段）

顶层状态机，决定当前应该执行什么策略。`play()` 函数根据 `context.game.phase` 分派到对应的 `_act_xxx()` 方法：

| Phase | 含义 | 策略行为 |
|-------|------|----------|
| `NORMAL` | 正常比赛 | 最近的人进攻，最近球门的人守门，其余支援 |
| `OUR_KICKOFF` | 己方开球 | 锁定开球人执行 take_kickoff，其余站到开球站位 |
| `OPP_KICKOFF` | 对方开球 | 一人守门，其余站到中圈外开球站位 |
| `OUR_SET_PLAY` | 己方定位球 | 按定位球类型分派（目前默认走 NORMAL） |
| `OPP_SET_PLAY` | 对方定位球 | 全员静止等待 |
| `READY` | 准备阶段 | 各自走到预设站位 |
| `STOPPED` | 比赛暂停 | 全员停车 |

### 5.3 Action（球员动作意图）

每个球员的 `action` 字段记录当前意图，用于调试和状态追踪。定义在 `types.py` 的 `Action` 枚举中：

```python
Action.INIT           # 初始化
Action.STOPPED        # 已停止
Action.ATTACK         # 进攻
Action.GUARD          # 防守
Action.SUPPORT        # 支援
Action.READY          # 准备阶段站位
Action.KICKOFF        # 己方开球站位
Action.OPP_KICKOFF_READY  # 对方开球站位
Action.STAY           # 停留
Action.PENALIZED      # 罚牌中
Action.FALLEN         # 摔倒
Action.SWITCHING_MODE # 切换模式中
Action.NO_POSE        # 无位姿信息
```

### 5.4 Player（球员实例）

每个 Player 对象跨整场比赛存活，`context` 每帧被框架覆写。Player 继承 `MotionMixin` 获得行走/踢球能力，自身定义高层动作。

**核心属性：**
```python
player.id           # 球员编号（1/2/3）
player.pose         # 位姿（x, y, theta）
player.is_fallen    # 是否摔倒
player.is_penalized # 是否被罚牌
player.is_kicking   # 是否正在踢球
player.action       # 当前动作意图（Action 枚举）
player.context      # 当前帧上下文（每帧注入）
```

## 6. 比赛阶段（Phase）详解

### 6.1 NORMAL — 正常比赛

最常用的阶段。策略逻辑：

```python
def _act_normal(context, players):
    # 1. 选择进攻者（离球最近，含粘性选择防震荡 + 摔倒惩罚）
    attacker = _select_closest_attacker(context, players)
    attacker.attack()
    
    # 2. 剩余人中离己方球门最近的守门
    rest = [p for p in players if p is not attacker]
    guard = min(rest, key=lambda p: dist(p.pose.x, p.pose.y, gx, gy))
    guard.guard()
    
    # 3. 其余人支援
    rest = [p for p in rest if p is not guard]
    for p in rest:
        p.support()
```

**粘性选择机制**：一旦某人被选为进攻者，在一定距离内保持不变（`ATTACKER_KEEP_DIST_MARGIN_M = 0.3m`），防止进攻者在球员之间来回切换产生震荡。

**场景示例**：
- 球在己方半场：进攻者追球，守门员守在球门区，支援者站在球与球门之间
- 球在对方半场：进攻者直接射门，守门员前移，支援者跟进

### 6.2 OUR_KICKOFF — 己方开球

```python
def _act_our_kickoff(context, players):
    # 1. 锁定开球人（进入阶段时选择，跨帧保持）
    if prev_phase != Phase.OUR_KICKOFF or state.kickoff_taker is None:
        state.kickoff_taker = _select_closest_attacker(context, players).id
    
    # 2. 开球人执行 take_kickoff（走到球后 + 踢球）
    attacker = next((p for p in players if p.id == state.kickoff_taker), None)
    attacker.take_kickoff()
    
    # 3. 其余人站到开球站位
    rest = [p for p in players if p is not attacker]
    Player.walk_to_slots(rest, OUR_KICKOFF_SLOTS, Action.KICKOFF)
```

**开球策略**：`take_kickoff()` 会先检查球员是否在球后方的正确位置，如果不在则走到球后方 `KICKOFF_STAGE_M = 2.0m` 处，到位后再执行踢球。

### 6.3 OPP_KICKOFF — 对方开球

```python
def _act_opp_kickoff(context, players):
    # 1. 一人守门
    guard = min(players, key=lambda p: dist(p.pose.x, p.pose.y, gx, gy))
    guard.guard()
    
    # 2. 其余人站到中圈外开球站位
    rest = [p for p in players if p is not guard]
    Player.walk_to_slots(rest, OPP_KICKOFF_SLOTS, Action.OPP_KICKOFF_READY)
```

**策略意图**：避免提前进入中圈违例，同时准备好反击。

### 6.4 READY — 准备阶段

比赛开始/重新开始前的站位阶段。根据是否己方开球，走不同的预设站位：

```python
def _act_ready(context, players):
    our_kickoff = game.kicking_team == context.team_id
    positions = READY_OUR_POSITIONS if our_kickoff else READY_OPP_POSITIONS
    Player.walk_to_slots(players, positions, Action.READY, face=0.0)
```

站位坐标在 `param.py` 中配置，按重要性从高到低排列（球员不足时优先站重要位置）：

```python
READY_OUR_POSITIONS = [
    (-6.0, 0.0),   # 球员1: 守门员（最重要）
    (-1.5, 0.0),   # 球员2: 中圈后方中心
    (-0.5, 3.5),   # 球员3: 中圈右侧
]

READY_OPP_POSITIONS = [
    (-6.0, 0.0),   # 球员1: 守门员（最重要）
    (-4.0, 0.0),   # 球员2: 禁区前沿中心
    (-2.0, 0.0),   # 球员3: 中圈后方中心
]
```

## 7. 球员动作详解

### 7.1 attack() — 进攻

```python
def attack(self, kick_target=None):
    # 默认射向对方球门中心
    kick_target = kick_target or context.field.opponent_goal
    
    # 计算踢球力度（后场/前场/开球阶段不同）
    power = KICK_POWER_BACKFIELD if ball.x < 0 else KICK_POWER_DEFAULT
    
    # 判断是否进入踢球状态
    d = dist(self.pose.x, self.pose.y, ball.x, ball.y)
    self._kicking = d <= (KICK_EXIT_M if self._kicking else KICK_ENTER_M)
    
    if self._kicking:
        # 距球够近，执行踢球
        kick_direction = angle_to(ball.x, ball.y, *kick_target)
        self.kick(kick_direction, power)
    else:
        # 距球较远，走到球后方准备位置
        self.walk_to(_behind_ball(ball.x, ball.y, kick_target, CHASE_BEHIND_M))
```

**踢球状态判定**：使用滞回机制（hysteresis），避免在边界处频繁切换：
- 进入踢球：距球 < `KICK_ENTER_M = 2.0m`
- 退出踢球：距球 > `KICK_EXIT_M = 2.5m`

**场景示例**：
- 近距离射门：直接计算方向踢球
- 远距离追球：走到球后方 `CHASE_BEHIND_M = 0.35m` 处，面向球门

### 7.2 guard() — 防守

```python
def guard(self):
    # 站位在己方球门区中心
    home = context.field.own_goal_area_center  # (-6.5, 0.0)
    
    # 可选：面向球
    if GUARD_FACE_BALL:
        face = angle_to(self.pose.x, self.pose.y, ball.x, ball.y)
    
    self.walk_to(home, face=face, avoid_ball=True, avoid_robots=True)
```

**策略意图**：守门员始终站在球门区中心，面向球准备扑救。

### 7.3 support() — 支援

```python
def support(self):
    # 站在球与己方球门之间的连线上
    # 距球不超过 SUPPORT_DIST_M = 3.0m
    ux, uy = normalize(own_goal_x - ball.x, own_goal_y - ball.y)
    along = min(SUPPORT_DIST_M, dist(ball.x, ball.y, own_goal_x, own_goal_y) - GUARD_DIST_M)
    tx = ball.x + ux * along
    ty = ball.y + uy * along
    
    # 面向球移动
    face = angle_to(self.pose.x, self.pose.y, ball.x, ball.y)
    self.walk_to((tx, ty), face=face, avoid_ball=True, avoid_robots=True)
```

**策略意图**：支援者站在球和球门之间，既可以协助防守，也可以在进攻者传球时接应。

### 7.4 take_kickoff() — 开球

```python
def take_kickoff(self, kick_target=None):
    # 默认射向对方球门
    kick_target = kick_target or context.field.opponent_goal
    
    # 计算踢球方向
    kick_dir = angle_to(ball.x, ball.y, *kick_target)
    
    # 计算球员相对于球的位置投影
    # behind: 沿踢球方向的距离（正=在球前方，负=在球后方）
    # lateral: 垂直踢球方向的偏差
    behind = rel_x * cos_k + rel_y * sin_k
    lateral = abs(-rel_x * sin_k + rel_y * cos_k)
    
    # 如果位置不在球后方合适区域，先走到球后方准备位置
    if behind > KICKOFF_FRONT_MARGIN or lateral > KICKOFF_LATERAL_TOL:
        stage = (ball.x - cos_k * KICKOFF_STAGE_M, ball.y - sin_k * KICKOFF_STAGE_M)
        self.walk_to(stage, face=kick_dir, avoid_ball=True)
    else:
        # 位置到位，执行踢球
        self.kick(kick_dir, KICK_POWER_OUR_KICKOFF)
```

**开球逻辑**：
1. 计算踢球方向（从球指向对方球门）
2. 判断球员是否在球后方的"踢球位"：
   - 沿踢球方向距离 < `KICKOFF_FRONT_MARGIN = 0.1m`（不能在球前方太远）
   - 侧向偏差 < `KICKOFF_LATERAL_TOL = 0.35m`（不能偏离踢球方向太远）
3. 如果不在正确位置，走到球后方 `KICKOFF_STAGE_M = 2.0m` 处
4. 位置到位后直接踢球

### 7.5 walk_to_slots() — 批量站位（静态方法）

```python
@staticmethod
def walk_to_slots(players, positions, action=Action.STAY, face=None):
    """将多个球员分配到指定站位，使用最优匹配使总移动距离最短。"""
    
    # 计算距离矩阵
    dist_matrix = [
        [dist(p.pose.x, p.pose.y, pos[0], pos[1]) for pos in positions]
        for p in players
    ]
    
    # 枚举所有排列，选择总距离最小的配对
    best_perm = None
    best_total = float("inf")
    for perm in permutations(range(n)):
        total = sum(dist_matrix[i][perm[i]] for i in range(n))
        if total < best_total:
            best_total = total
            best_perm = perm
    
    # 按最优配对分配站位
    for i, pos_idx in enumerate(best_perm):
        players[i].walk_to(positions[pos_idx], ...)
```

**设计亮点**：
- 使用最优匹配算法（穷举排列）使总移动距离最短
- 支持临时坐标列表作为参数
- positions 按重要性从高到低排列，球员不足时优先站重要位置
- 适用于 READY、KICKOFF 等需要多人站位的场景

## 8. 参数调节指南

所有可调参数集中在 `src/param.py`，以下是关键参数：

### 8.1 踢球力度

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `KICK_POWER_MIN` | 1.0 | 最小踢球力度 |
| `KICK_POWER_MAX` | 10.0 | 最大踢球力度 |
| `KICK_POWER_DEFAULT` | 5.0 | 普通比赛踢球力度 |
| `KICK_POWER_BACKFIELD` | 5.0 | 后场踢球力度 |
| `KICK_POWER_OUR_KICKOFF` | 5.0 | 开球踢球力度 |

**调参建议**：
- 力度太小：球踢不远，容易被对方拦截
- 力度太大：球容易出界，或传球不准
- 开球力度建议调大（如 6.0-7.0），争取一开球就射门

### 8.2 行走控制

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `ARRIVE_DIST` | 0.15m | 到达目标点阈值 |
| `OMNI_DIST` | 1.0m | 小于此距离用全向近距控制 |
| `TURN_THRESHOLD` | 0.5 rad | 角差超过此值先原地转向 |
| `MAX_LINEAR` | 2.0 m/s | 前进速度上限 |
| `MAX_ANGULAR` | 2.0 rad/s | 转向速度上限 |
| `LINEAR_GAIN` | 1.5 | 平移 P 增益 |
| `ANGULAR_GAIN` | 2.0 | 转向 P 增益 |

**调参建议**：
- `ARRIVE_DIST` 太小：球员会在目标点附近抖动
- `LINEAR_GAIN` 太大：球员反应快但容易过冲
- `ANGULAR_GAIN` 太大：转向太快，球员不稳定

### 8.3 踢球状态判定

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `KICK_ENTER_M` | 2.0m | 距球小于此值进入踢球状态 |
| `KICK_EXIT_M` | 2.5m | 踢球中距球大于此值才退出 |
| `CHASE_BEHIND_M` | 0.35m | 追球时站到球后方的距离 |

**调参建议**：
- `KICK_ENTER_M` 太大：球员离球很远就开始尝试踢球，效果差

## 9. 避障与路径规划

### 9.1 障碍物定义

框架将以下对象视为障碍：

| 障碍类型 | 半径 | 说明 |
|----------|------|------|
| 球 | `BALL_OBSTACLE_RADIUS = 0.5m` | 对方开球时球也是障碍 |
| 对手机器人 | `OPPONENT_RADIUS = 0.55m` | |
| 队友机器人 | `TEAMMATE_RADIUS = 0.48m` | 排除自己 |
| 球门结构 | `POST_RADIUS = 0.18m` | 门柱 + 球网采样 |

### 9.2 A* 全局路径规划

`path_planner.py` 实现了基于栅格的 A* 算法：

1. 在场地内建立 8 邻接栅格（分辨率 0.35m）
2. 将障碍物标记为不可通行
3. 从起点到终点搜索最短路径
4. 对路径进行平滑处理
5. 提取前方 `GLOBAL_PATH_LOOKAHEAD_M = 0.9m` 处的路径点作为行走目标

### 9.3 VFH 局部规划

当全局规划不可用时，回退到 VFH（Vector Field Histogram）局部规划：

1. 从当前位置向目标方向发射探测射线
2. 在 `±PLAN_MAX_OFFSET` 范围内扫描候选方向
3. 选择安全余量 `≥PLAN_CLEARANCE` 的最优方向
4. 奇偶 ID 球员扫描方向相反，避免集体撞墙

### 9.4 walk_to() 中的避障

```python
def walk_to(target, avoid_ball=False, avoid_robots=False):
    if 需要避障:
        obstacles = collect_obstacles(context, ...)
        if 启用全局规划:
            path = plan_global_path(start, target, obstacles)
            waypoint = 提取前方路径点(path)
            heading = 指向路径点
        else:
            heading = VFH局部规划(goal_dir, obstacles)
    else:
        heading = 直线指向目标
    
    # 根据距离选择控制方式
    if 距离 < OMNI_DIST:
        全向近距控制(vx, vy, vyaw)
    else:
        先转向再前进(vx, 0, vyaw)
```

### 10.1 场景一：调整开球站位

**想法**：开球时，我想让两个非开球者站得更靠前一些，以便快速反击。

**实现步骤**：

1. **修改 `param.py`**：调整 `OUR_KICKOFF_SLOTS` 的坐标
   ```python
   OUR_KICKOFF_SLOTS = [
       (-2.0, 0.5),   # 更靠前
       (-2.0, -0.5),  # 更靠前
   ]
   ```

2. **验证**：开球阶段，非开球者会站到新的位置

### 10.2 场景二：改变进攻策略

**想法**：正常比赛时，我不想让离球最近的人进攻，而是固定让 1 号球员进攻，2 号守门员，3 号支援。

**实现步骤**：

1. **修改 `main.py` 的 `_act_normal()`**：
   ```python
   def _act_normal(context: Context, players: list[Player]) -> None:
       if not players:
           return
       
       # 固定角色分配
       attacker = next((p for p in players if p.id == 1), players[0])
       guard = next((p for p in players if p.id == 2), players[1] if len(players) > 1 else players[0])
       support = next((p for p in players if p.id == 3), players[2] if len(players) > 2 else None)
       
       attacker.attack()
       guard.guard()
       if support:
           support.support()
   ```

2. **验证**：无论球在哪里，1号始终进攻，2号始终守门，3号始终支援

### 10.3 场景三：添加新动作

**想法**：我想添加一个"拦截"动作，让球员站在球和对方球门之间。

**实现步骤**：

1. **在 `types.py` 的 `Action` 枚举中添加新动作**：
   ```python
   INTERCEPT = "intercept"
   ```

2. **在 `player.py` 中添加新方法**：
   ```python
   def intercept(self):
       """拦截动作:站在球与对方球门之间。"""
       self.action = Action.INTERCEPT
       ball = self.context.ball
       if ball is None:
           self.stop()
           return
       
       # 计算球到对方球门的方向
       opp_goal = self.context.field.opponent_goal
       ux, uy = normalize(opp_goal[0] - ball.x, opp_goal[1] - ball.y)
       
       # 站在球前方一段距离
       intercept_dist = 2.0  # 可以放到 param.py 中
       tx = ball.x + ux * intercept_dist
       ty = ball.y + uy * intercept_dist
       
       # 面向球
       face = angle_to(self.pose.x, self.pose.y, ball.x, ball.y)
       self.walk_to((tx, ty), face=face, avoid_ball=True, avoid_robots=True)
   ```

3. **在 `main.py` 中使用新动作**：
   ```python
   # 在 _act_normal 中替换 support
   for p in rest:
       p.intercept()
   ```

4. **验证**：支援者会站在球和对方球门之间，准备拦截传球

### 10.4 场景四：修改射门目标

**想法**：我想让球员射向球门的右上角，而不是球门中心。

**实现步骤**：

1. **在 `param.py` 中添加新参数**：
   ```python
   # 射门目标偏移（相对于球门中心）
   KICK_TARGET_OFFSET_X = 0.0
   KICK_TARGET_OFFSET_Y = 2.0  # 向上偏移 2m，即球门右上角
   ```

2. **修改 `player.py` 的 `attack()` 方法**：
   ```python
   def attack(self, kick_target=None):
       self.action = Action.ATTACK
       
       # 默认射向对方球门右上角
       opp_goal = self.context.field.opponent_goal
       kick_target = kick_target or (opp_goal[0] + KICK_TARGET_OFFSET_X, opp_goal[1] + KICK_TARGET_OFFSET_Y)
       
       # 其余逻辑不变...
   ```

3. **验证**：球员会射向球门右上角

### 10.5 场景五：添加新的比赛阶段处理

**想法**：我想在 CORNER_KICK（角球）阶段实现特殊的角球战术。

**实现步骤**：

1. **修改 `main.py` 的 `_act_our_set_play()`**：
   ```python
   def _act_our_set_play(context: Context, players: list[Player]) -> None:
       context.game.strategy_state.normal_attacker = None
       set_play = get_set_play_type(context)
       
       if set_play == SetType.CORNER_KICK:
           _act_corner_kick(context, players)
           return
       
       # 其他情况走正常逻辑
       _act_normal(context, players)
   ```

2. **添加新的处理函数**：
   ```python
   def _act_corner_kick(context: Context, players: list[Player]) -> None:
       """角球战术:一人发球,两人冲到禁区抢点。"""
       if not players:
           return
       
       # 选择发球者（离球最近的）
       ball = context.ball
       kicker = min(players, key=lambda p: dist(p.pose.x, p.pose.y, ball.x, ball.y))
       kicker.take_kickoff()
       
       # 其余人冲到禁区抢点
       rest = [p for p in players if p is not kicker]
       # 禁区抢点位置
       corner_slots = [
           (10.0, 1.0),   # 禁区内左侧
           (10.0, -1.0),  # 禁区内右侧
       ]
       Player.walk_to_slots(rest, corner_slots, Action.ATTACK)
   ```

3. **验证**：角球时，一人发球，其余人冲进禁区抢点

### 10.6 场景六：使用临时站位

**想法**：在某个特殊情况下，我想让球员站到临时指定的位置。

**实现步骤**：

```python
# 在 main.py 中直接传入临时坐标列表
temp_slots = [
    (5.0, 2.0),
    (5.0, -2.0),
]

```

## 常见问题

### Q1：为什么开球时球员不踢球？

**可能原因**：
1. 球员不在球后方的正确位置（`take_kickoff` 会先走到球后方）
2. `attack()` 方法参数错误（之前的 bug：传入 `target` 而不是 `kick_direction`）
3. `KICK_POWER_OUR_KICKOFF` 太小

**排查方法**：
- 查看 debugdraw 显示的球员动作标签，确认球员是否执行了 `take_kickoff`
- 检查球员是否走到了球后方位置
- 检查日志是否有错误信息

### Q2：如何让球员面向特定方向？

**方法**：调用 `walk_to()` 时传入 `face` 参数：
```python
# 面向球
face = angle_to(self.pose.x, self.pose.y, ball.x, ball.y)
self.walk_to(target, face=face)

# 面向固定方向（如朝向对方球门）
self.walk_to(target, face=0.0)  # 0.0 = 正前方
```

### Q3：如何实现跨帧状态记忆？

**方法**：使用 `context.game.strategy_state`：
```python
# 设置状态
context.game.strategy_state.my_custom_state = "value"

# 读取状态（跨帧保持）
value = context.game.strategy_state.my_custom_state
```

### Q4：如何判断球员是否到达目标位置？

**方法**：`walk_to()` 返回 `bool`，`True` 表示已到达：
```python
if self.walk_to(target):
    # 已到达目标
    self.stop()
```

### Q5：如何自定义踢球规划？

**方法**：`kick()` 方法在 `kick_direction=None` 时会自动调用 `plan_kick()`。如果需要自定义规划，可以在调用前计算好方向：
```python
kick_direction = angle_to(ball.x, ball.y, target_x, target_y)
self.kick(kick_direction, power=5.0)
```

```
│   │       └── players: tuple[PlayerState, ...]
│   │           └── PlayerState
│   │               ├── penalty: Penalty        # 罚牌类型
│   │               └── secs_till_unpenalised: int  # 剩余罚时
│   ├── .get_team_state(team_id) -> TeamState | None
│   ├── .get_player_state(team_id, player_id) -> PlayerState | None
│   └── ._compute_phase(team_id) -> Phase       # 内部:计算比赛阶段
│
├── teammates: dict[int, RobotState]    # 队友字典 {player_id: RobotState}
│   └── RobotState
│       ├── player_id: int              #   球员编号
│       ├── pose: Pose2D | None         #   位姿 (可能为 None!)
│       │   ├── x: float                #     x 坐标
│       │   ├── y: float                #     y 坐标
│       │   └── theta: float            #     朝向角度 (弧度)
│       └── last_seen_at: float         #   最后观测时间
│
├── opponents: dict[int, RobotState]    # 对手字典 {player_id: RobotState}
│   └── (结构同 teammates)
│
└── pre_context: Context | None         # 上一帧的 Context (可用于差分计算)
```

### 如何使用 Context

#### 核心原则：Context 是只读快照

`Context` 是 `frozen=True` 的 dataclass，**所有字段都是只读的**。框架每帧构造一个新的 `Context` 传给 `play()` 函数，你的策略代码基于这个快照做决策，不要试图修改它。

#### 安全访问模式

由于 `ball`、`game` 可能为 `None`，访问前**必须做空值检查**：

```python
def play(self, context: Context, players: list) -> list:
    # 1. 安全访问球
    ball = context.ball
    if ball is None:
        # 没有球的信息，执行降级策略
        return [(Action.STAY,)] * 3
    
    # 此时 ball 一定有值，可以安全使用
    ball_x, ball_y = ball.x, ball.y

    # 2. 安全访问比赛状态
    if context.game is not None:
        phase = context.game.phase        # Phase.NORMAL / Phase.OUR_KICKOFF 等
        state = context.game.state        # GameState.PLAYING / GameState.READY 等
        set_play = context.game.set_play  # SetType.DIRECT_FREE_KICK 等

    # 3. 安全访问队友/对手
    teammate1 = context.teammates.get(1)  # 用 .get() 避免 KeyError
    if teammate1 and teammate1.pose:
        tx, ty = teammate1.pose.x, teammate1.pose.y
    
    for opp_id, opp in context.opponents.items():
        if opp.pose:
            # 处理每个对手的位置
            pass

    # 4. 场地信息始终可用
    opponent_goal = context.field.opponent_goal  # (x, y) 元组
    own_goal = context.field.own_goal
    half_length = context.field.half_length
```

#### 典型使用场景

**根据比赛阶段选择策略：**

```python
if context.game and context.game.phase == Phase.OUR_KICKOFF:
    # 我方开球 → 执行开球策略
    ...
elif context.game and context.game.phase == Phase.OPP_SET_PLAY:
    # 对方定位球 → 防守站位
    ...
else:
    # 正常拼抢
    ...
```

**利用 `pre_context` 做差分/速度估算：**

```python
if context.pre_context and context.pre_context.ball and context.ball:
    prev_ball = context.pre_context.ball
    # 估算球的速度方向
    ball_vx = (context.ball.x - prev_ball.x) / context.dt
    ball_vy = (context.ball.y - prev_ball.y) / context.dt
```

**利用 `strategy_state` 跨帧传递状态：**

`strategy_state` 是一个 `SimpleNamespace`，可以往上面挂任意属性，框架会在帧间保留：

```python
# 第一帧记录某个状态
context.game.strategy_state.kick_target = (7.0, 1.2)

# 后续帧读取
saved_target = getattr(context.game.strategy_state, "kick_target", None)
```
